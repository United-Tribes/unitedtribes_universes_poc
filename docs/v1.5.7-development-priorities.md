## Priority 1: Modal Standardization

**This is the #1 design priority.** Every modal and popover must follow consistent rules.

### Universal Modal Rules
1. **No external links. Ever.** No "Watch on YouTube ↗", no `target="_blank"`, no arrows that open new tabs. Everything plays inline in our modals.
2. **"Add to My Stuff" on everything.** Every modal, every popover, every piece of surfaced content gets a save-to-library button. No exceptions.
3. **Inline playback.** Videos play in VideoModal. Podcasts play in PodcastModal. Nothing leaves the app.
4. **Timestamp references.** When we cite a source, point to the specific moment — not just the video/podcast generically. "At 14:32, Gilligan discusses..." with a play-from-here button.

### Modals to Standardize
| Modal | Current State | Needed |
|-------|--------------|--------|
| **VideoModal** | Opens for YouTube videos, full-width | Add "Add to My Stuff", ensure no external links |
| **PodcastModal** | Basic HTML5 player, no artwork | Correct artwork, forward/back controls, analysis highlights with timestamps |
| **SourcePopover** | Shows on citation click, "Watch on YouTube" button | Play inline, add-to-library, timestamp reference |
| **Entity Popover** | Shows entity info + Ask input | Add-to-library, fix type badges, contextual media, no dead ends |

## The Treasure Trove Principle — CRITICAL

We have an amazing analyzed media collection — 582 videos, 47 podcasts, 8,435+ Pluribus relationships, timestamps, key moments, entity mentions, quotes, and **discovery playlists for every analyzed piece of media.** This is only valuable if we connect the dots:

- A citation should point to "at 14:32 in this interview, Gilligan explains X" — not just "here's a video about Pluribus"
- The harvester has already done the analysis. The timestamps and key moments exist in the data.
- Every surface in the UI (citations, popovers, modals, episode pages, cast pages) should be pulling from this analyzed data to show specific, actionable moments
- The competitive advantage is showing users exactly WHERE in a piece of media the thing they care about is discussed
- **Discovery playlists from analyzed media need to be aggregated** — each video/podcast analysis produces discovery playlists (related content, themes, entities). These playlists, understood in aggregate, are the fuel for the Discover tab. Without understanding what's IN them, the Discover tab is empty.
- **The Discover tab is a priority.** It should be powered by the aggregated discovery playlists and analysis from all processed media — not generic recommendations, but specific, grounded connections derived from what the harvester has actually found in the content.
