# Cast & Creators — Complete Design Briefing

> Generated Feb 27, 2026 from live codebase analysis + broker API query

---

## 1. Screens & Views for Cast & Creators

When a user clicks **"Cast & Creators"** in the SideNav, they enter the `CastCrewScreen` component, which has **3 internal views** controlled by a `view` state variable:

### View 1: Lobby (`view === "lobby"`)
The default landing page showing an overview of all cast and creators.

**What the user sees (top to bottom):**
- **Header row:** "◉ Cast & Creators" title + "{N} people" pill (opens/closes People Directory drawer)
- **Intro text:** "Pluribus reunites Vince Gilligan's trusted creative family..."
- **Stats line:** "Powered by UnitedTribes Knowledge Graph · 18 cast · 6 creators & crew"
- **Creator Spotlight:** Vince Gilligan card with photo, name, role, bio snippet, purple "CREATOR" badge, clickable
- **Timeline strip:** Horizontal bar showing X-Files → Breaking Bad → Better Call Saul → El Camino → Pluribus with role/year under each. Pluribus is highlighted blue. Other shows are clickable (navigate to entity detail).
- **The Cast** section: 3-column grid of `CastCard` components (29 cards). Each card shows: photo (100×130px), actor name, character name in blue, type badge (LEAD = blue, CAST = purple), video count badge.
- **Key Crew** section: Vertical list of `CrewRow` components. Each row shows: photo (44×44px), name, role context, purple "CREW" badge. Excludes Vince Gilligan, BTR1, and Ricky Cook.

### View 2: Cast Detail (`view === "castDetail"`)
A full profile page for an individual actor. Reached by clicking a CastCard in the lobby or a name in the People Directory drawer.

**What the user sees (top to bottom):**
1. **Hero:** Photo (160×200px) + actor name (28px heading) + "as [Character]" in blue + entity tags (birthdate, hometown, field) + context sentence
2. **In Pluribus — [Character]:** (Only shows if character entity exists in KG) Single paragraph character description. Red accent bar on left.
3. **Character Themes:** (NEVER renders currently — `charEntity.themes` is always empty array)
4. **About [Actor Name]:** Blue accent bar. Dynamic bio generated by broker API or loaded from cache. Has "Regenerate" and "Edit" toolbar. Falls back to entity bio from TMDB if broker unavailable.
5. **Interviews & Appearances:** Red accent bar. Up to 4 YouTube video embeds from entity's `quickViewGroups` Videos + `interviews` data. Each is a full iframe embed.
6. **Known For:** Green accent bar. Up to 10 items from `completeWorks`. Each row shows: type badge (TV/FILM), optional poster thumbnail (36×52px), title, role·year meta. Gilligan-universe works get gold border + "GILLIGAN" badge.
7. **Collaborators:** Cyan accent bar. Up to 8 collaborators in 2-column grid. Each shows: avatar (36px), name, relationship description, "In Cast" badge if they're in the show's cast. Clickable to navigate to their detail page.

**What's NOT on this page:** No "Add to Library" buttons anywhere. No "Watch"/"Stream" CTAs. No articles. No inspirations. No music/sonic section. No mini constellation. No purchase links.

### View 3: Crew Detail (`view === "crewDetail"`)
A profile page for crew members (writers, directors, producers, composers). Reached by clicking a CrewRow or Vince Gilligan spotlight.

**What the user sees (top to bottom):**
1. **Hero:** Photo (160×200px) + name (28px heading) + purple role badge (e.g. "CREW") + context sentence. No character name (crew don't play characters).
2. **About [Crew Name]:** Blue accent bar. Same broker API bio with same cache/edit toolbar.
3. **Videos:** Red accent bar. Up to 4 YouTube embeds (same as cast).
4. **Known For:** Green accent bar. Up to 10 `completeWorks` items with same layout.

**What's NOT on this page:** No collaborators section. No articles. No inspirations. No sonic. No library buttons. No CTAs.

### People Directory Drawer (overlay, any view)
A 400px right-side drawer that persists across view changes. Toggled by the "{N} people" pill in lobby or the "PEOPLE" tab on the right edge.

**Sections:** Cast (18) tab, Creators & Crew (6) tab, "← Overview" back link (on detail views only)
**Content:** Lead Cast (8 rows) → Cast (10 rows) → Creators & Key Crew (6 rows). Each row: 56px photo, name, character/role, gold highlight for active person.

## 2. What Data Drives These Pages

### Data Source 1: `pluribus-response.json` (static, loaded at startup)
- `discoveryGroups[1].cards` — 29 cast card objects (title, meta, context, type, image_url)
- `discoveryGroups[2].cards` — 8 crew card objects (same shape)
- `actorCharacterMap` — 157 actor→character name mappings (e.g., `"Rhea Seehorn": "Carol Sturka"`)
- `themeVideos` — theme-to-video mappings with character moments (NOT currently used on cast pages)
- `songs` — 34 needle drops (NOT currently used on cast pages)

### Data Source 2: `pluribus-universe.json` (static, loaded at startup)
56 entities keyed by name. This is the rich entity data with full profiles.

### Data Source 3: Broker API (dynamic, on-demand)
`POST /v2/broker` with query like "Tell me about [person] in the context of Pluribus" — generates narrative bios.

**Props:** `onNavigate, onSelectEntity, library, toggleLibrary, selectedModel, onModelChange, entities, responseData, selectedUniverse, onUniverseChange, onNewChat, hasActiveResponse`

### CrewRow (line ~5706)
**Props:** `person` (object with title, type, photoUrl, context, repertory), `onEntityClick`
Renders: Photo (44×44px), name, context/role, CREW badge.

### ThemePill (line ~197)
**Props:** `themeId`, `onClick`, `size`
Renders: Colored pill for themes like "grief", "identity", etc. Used in character themes section.

### renderCastDetail() (line 6849) — internal function, not a component

### renderCrewDetail() (line 7057) — internal function, not a component

### renderPeopleDrawer() (line 7334) — internal function

### Lobby View
**Layout:** Single scrollable column, max-width 820px, left-aligned with 48px padding. SideNav (72px) on left, TopNav bar at top. Optional People Directory drawer (400px) on right.

**Top section:** Large "Cast & Creators" heading with warm circle icon. Gold-bordered pill showing total people count. Stats line underneath.

**Creator Spotlight:** A card-like element with Vince Gilligan's photo, name, "Creator, Showrunner..." role, purple CREATOR badge, and a one-line bio snippet. Below it, a horizontal timeline bar showing his career progression from X-Files to Pluribus.

**Cast grid:** 3 columns of cards. Each card has a TMDB headshot at top, actor name in bold below, character name in blue text, and a type badge (LEAD in blue or CAST in purple). Cards are clickable — navigate to cast detail.

**Crew section:** Simpler vertical list. Each row has a small round photo, name, role description, and purple CREW badge. Clicking navigates to crew detail.

**What works:** Navigation between lobby and detail views. People drawer opens/closes with animation. Bio generation and caching. Video embeds play.
**What doesn't work:** Character themes section never renders (empty data). No library integration. No external links. Limited video display (4 max). Large amounts of entity data go unused.

### Cast Detail View (e.g., Rhea Seehorn)
**Layout:** Same single column, max-width 820px. People drawer can be open alongside.

**Hero:** Left-aligned photo (tall rectangle) next to stacked info. Name is large (28px). Character name ("as Carol Sturka") in blue below. Small gray tag pills for birth date, hometown, field. One-sentence context below.

**Character section:** Only appears for characters with entities in the KG (5 of 18 cast). Shows a single paragraph describing the character's role. Red bar on left.

**Bio section:** "About Rhea Seehorn" with blue bar. Shows either a broker-generated 2-3 paragraph bio (warm editorial tone, auto-generated from KG data) or falls back to the TMDB Wikipedia bio. Small toolbar below for "Regenerate" and "Edit".

**Videos:** "Interviews & Appearances" with red bar. Up to 4 YouTube iframe embeds in a responsive grid (1 col if 1 video, 2 cols if 2+). Full embedded players, not thumbnails.

**Known For:** "Known For" with green bar. Vertical list of up to 10 works. Each row: small poster thumbnail, type badge (TV/FILM), title, role/year. Gilligan-universe works highlighted with gold border and "GILLIGAN" text badge.

**Collaborators:** "Collaborators" with cyan bar. 2-column grid of small profile chips. Each has a round avatar, name, role, and "In Cast" badge if applicable. Clickable.

### Crew Detail View (e.g., Vince Gilligan)
**Layout:** Same as cast detail but simpler.

**Hero:** Same layout but no character name. Shows purple role badge instead.

**Bio:** "About Vince Gilligan" — same broker API bio system.

**Videos:** "Videos" — same YouTube embeds, max 4.

**Known For:** Same as cast. Shows filmography with Gilligan highlights.

**No collaborators section.** Page ends after Known For.
